# 3.4: Web UI HTTP handlers

## Description

在 `internal/handler/` 实现所有 Web UI 端点的 HTTP handler，挂载到 `/api/` 前缀。

**需实现的端点：**

| Method | Path | Handler |
|--------|------|---------|
| GET | `/api/projects` | `ListProjects` |
| GET | `/api/projects/{id}` | `GetProject` |
| GET | `/api/features/{id}/tasks` | `GetFeatureTasks` |
| GET | `/api/tasks/{id}` | `GetTask` |
| GET | `/api/tasks/{id}/records` | `ListTaskRecords` |
| GET | `/api/proposals/{id}/content` | `GetProposalContent` |
| GET | `/api/features/{id}/content` | `GetFeatureContent` |

**统一错误处理：**
```go
func writeError(w http.ResponseWriter, err error) {
    switch {
    case errors.Is(err, ErrNotFound):
        respondJSON(w, 404, errorBody("not_found", ...))
    case errors.Is(err, ErrNoAvailableTask):
        respondJSON(w, 404, errorBody("no_available_task", ...))
    // ... 其他 sentinel errors
    default:
        respondJSON(w, 500, errorBody("internal_error", ...))
    }
}
```

**GetFeatureTasks** 需从 query string 解析 `priority`、`tag`、`status`（逗号分隔）构建 `TaskFilter`。

## Acceptance Criteria

- [ ] 所有端点返回正确的 JSON 结构（与 api-handbook 一致）
- [ ] `GET /api/projects?search=foo&page=1&pageSize=20` 参数正确传递
- [ ] `GET /api/features/{id}/tasks?priority=P0,P1&tag=core&status=pending` 过滤正确
- [ ] 不存在的资源返回 `{"error":"not_found","message":"...","hint":"..."}`
- [ ] `go test ./internal/handler/... -v` 使用 httptest，覆盖率 ≥ 60%

## Implementation Notes

- 使用 `chi.URLParam(r, "id")` 提取路径参数，`strconv.ParseInt` 转换
- `respondJSON` 辅助函数：设置 `Content-Type: application/json`，写入状态码和 JSON body
- `errorBody` 辅助函数：构建 `{"error":code,"message":msg,"hint":hint}` 结构
- handler 通过构造函数注入 service 依赖

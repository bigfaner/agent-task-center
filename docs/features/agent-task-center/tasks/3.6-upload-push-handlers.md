# 3.6: Upload & Push HTTP handlers

## Description

在 `internal/handler/` 实现文件上传和批量推送端点。

**需实现的端点：**

| Method | Path | Handler |
|--------|------|---------|
| POST | `/api/upload` | `UploadFile` |
| POST | `/api/push` | `PushDocs` |

**UploadFile：**
- `Content-Type: multipart/form-data`
- Query params: `project`（必填）、`feature`（上传 index.json 时必填）
- 文件字段名：`file`
- 文件大小限制：5MB（`r.ParseMultipartForm(5 << 20)`）
- 仅接受 `.json` 和 `.md` 文件，否则返回 400 `invalid_file`
- 调用 `UploadService.ParseAndUpsert`，返回 `UpsertSummary`

**PushDocs：**
- `Content-Type: multipart/form-data`
- Query param: `project`（必填）
- 文件字段名：`files`（多文件）
- 从每个文件的 `Filename` 提取路径信息，推断 featureSlug
- 调用 `UploadService.ParseAndUpsert` 逐一处理
- 返回所有文件的 `UpsertSummary` 列表 + 汇总统计

## Acceptance Criteria

- [ ] `POST /api/upload` 缺少 `project` 参数返回 400 `missing_field`
- [ ] 上传超过 5MB 文件返回 413 `file_too_large`
- [ ] 上传非 .json/.md 文件返回 400 `invalid_file`
- [ ] 成功上传返回正确的 `UpsertSummary` JSON
- [ ] `POST /api/push` 批量处理多文件，返回 `results` 数组 + `totalCreated`/`totalUpdated`

## Implementation Notes

- 文件大小检查：先调用 `r.ParseMultipartForm(5 << 20)`，超限会返回错误
- 文件扩展名检查：`filepath.Ext(header.Filename)`
- `PushDocs` 中路径解析：`docs/features/<slug>/tasks/index.json` → featureSlug = `<slug>`
- 路径遍历防护：校验文件名不包含 `..`

# 5.3: Web UI component tests

## Description

使用 Vitest + React Testing Library 为核心页面组件编写测试，API 调用使用 MSW mock。

**测试范围：**

| 组件/页面 | 测试用例 |
|---------|---------|
| `ProjectListPage` | 加载状态显示骨架屏；数据渲染正确；搜索过滤触发请求 |
| `FeatureKanbanPage` | 四列正确分组；筛选器变更触发 API 请求；URL 参数同步 |
| `TaskDetailPage` | 基本信息渲染；执行记录展开/折叠；无记录时空状态 |
| `StatusBadge` | 各状态值渲染正确颜色 |
| `PriorityBadge` | P0/P1/P2 渲染正确颜色 |
| `UploadDialog` | 文件校验错误提示；上传成功显示摘要 |

**MSW 配置（`src/mocks/handlers.ts`）：**
```typescript
import { http, HttpResponse } from 'msw'

export const handlers = [
  http.get('/api/projects', () => HttpResponse.json({ items: [...], total: 2 })),
  http.get('/api/features/:id/tasks', () => HttpResponse.json({ tasks: [...] })),
  // ...
]
```

**测试工具：**
- `vitest` — 测试运行器
- `@testing-library/react` — 组件渲染
- `@testing-library/user-event` — 用户交互模拟
- `msw` — API mock

## Implementation Notes

- MSW setup 放在 `src/mocks/`，`vitest.setup.ts` 中启动 MSW server
- 使用 `renderWithProviders` 辅助函数封装 QueryClient + Router provider
- 测试文件与组件同目录，命名 `*.test.tsx`
- `@testing-library/jest-dom` 提供 `toBeInTheDocument` 等 matcher

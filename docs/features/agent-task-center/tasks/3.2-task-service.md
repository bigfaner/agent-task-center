# 3.2: TaskService (claim / status / record)

## Description

在 `internal/service/` 实现 `TaskService`，包含任务认领（乐观锁）、状态更新、执行记录提交三个核心操作。

**TaskService 接口：**
```go
type TaskService interface {
    Get(ctx context.Context, id int64) (*TaskDetail, error)
    GetByTaskID(ctx context.Context, projectName, featureSlug, taskID string) (*TaskDetail, error)
    ListRecords(ctx context.Context, taskID int64, page, pageSize int) ([]ExecutionRecord, int, error)
    Claim(ctx context.Context, projectName, featureSlug, agentID string) (*Task, error)
    UpdateStatus(ctx context.Context, taskID int64, agentID, status string) error
    SubmitRecord(ctx context.Context, taskID int64, agentID string, record ExecutionRecord) (*ExecutionRecord, error)
}
```

**Claim 流程：**
1. 通过 `projectName` + `featureSlug` 定位 Feature
2. 调用 `db.ClaimTask`（乐观锁，最多重试 3 次）
3. 返回认领成功的 Task，或 `ErrNoAvailableTask`

**SubmitRecord：**
- 验证 `agentID` 与 `task.claimed_by` 一致，否则返回 `ErrUnauthorizedAgent`
- 插入 `ExecutionRecord`
- 将任务状态更新为 `completed`

## Acceptance Criteria

- [ ] `Claim` 在无可用任务时返回 `ErrNoAvailableTask`
- [ ] `UpdateStatus` 在 agentID 不匹配时返回 `ErrUnauthorizedAgent`
- [ ] `SubmitRecord` 成功后任务状态变为 `completed`
- [ ] `go test ./internal/service/... -v` 覆盖 Claim 乐观锁场景，覆盖率 ≥ 80%

## Implementation Notes

- `UpdateStatus` 合法状态值：`in_progress`、`blocked`、`pending`；其他值返回 `ErrInvalidStatus`（新增 sentinel error）
- `SubmitRecord` 中的 agentID 校验在 Service 层做，不依赖 DB 层
- `GetByTaskID` 用于 Agent API 的 `get-content` 端点，通过 project+feature+taskId 三元组定位

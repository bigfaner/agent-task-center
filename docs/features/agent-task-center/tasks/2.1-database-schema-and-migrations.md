# 2.1: Database schema & migrations

## Description

使用 `golang-migrate/migrate/v4` 创建数据库迁移文件，实现五张表的 DDL，并在服务启动时自动执行迁移。

**五张表：** `projects`, `proposals`, `features`, `tasks`, `execution_records`

迁移文件放在 `server/internal/db/migrations/`，使用 embed.FS 嵌入：
```
migrations/
├── 000001_init_schema.up.sql
└── 000001_init_schema.down.sql
```

启动时调用 `db.RunMigrations(db, migrationsFS)` 自动升级到最新版本。

## Acceptance Criteria

- [ ] SQLite 模式下 `go run ./cmd/server` 启动后五张表存在
- [ ] PostgreSQL 兼容：`BIGSERIAL PRIMARY KEY`、`TIMESTAMPTZ`
- [ ] down migration 可回滚（删除所有表）
- [ ] 重复启动不报错（migrate 幂等）
- [ ] `go test ./internal/db/...` 使用 in-memory SQLite 验证 migration 成功

## Implementation Notes

- SQLite DDL 使用 `INTEGER PRIMARY KEY AUTOINCREMENT`，PostgreSQL 使用 `BIGSERIAL PRIMARY KEY`
- 通过 `//go:embed migrations/*.sql` 嵌入迁移文件
- `tasks` 表的 `tags`、`dependencies` 字段存储 JSON 字符串（`TEXT NOT NULL DEFAULT '[]'`）
- `version` 字段初始为 0，用于乐观锁

# 3.7: Server main.go (wire up + embed.FS)

## Description

实现 `cmd/server/main.go`，完成所有组件的依赖注入、路由注册、静态文件服务，以及 Web UI 的 `embed.FS` 嵌入。

**启动流程：**
1. `config.Load()` 读取配置
2. `db.Connect(cfg)` 建立数据库连接
3. `db.RunMigrations(db, migrationsFS)` 执行迁移
4. 初始化 services（ProjectService、FeatureService、TaskService、UploadService）
5. 初始化 handlers，注册路由
6. 启动 HTTP server

**路由结构（使用 chi）：**
```go
r := chi.NewRouter()
r.Use(middleware.Logger)
r.Use(middleware.Recoverer)
r.Use(corsMiddleware)  // 开发时允许所有来源

// API 路由
r.Route("/api", func(r chi.Router) {
    r.Get("/projects", ...)
    r.Get("/projects/{id}", ...)
    // ... 其他 web UI 端点
    r.Post("/upload", ...)
    r.Post("/push", ...)
    r.Route("/agent", func(r chi.Router) {
        r.Post("/claim", ...)
        // ... 其他 agent 端点
    })
})

// 静态文件服务（embed.FS）
r.Handle("/*", staticHandler)
```

**embed.FS：**
```go
//go:embed web/dist
var webFS embed.FS

// SPA fallback：所有非 /api/ 请求返回 index.html
```

## Implementation Notes

- `embed.FS` 路径为 `web/dist`，需在 `server/` 目录下执行 `go build`（web 先构建）
- SPA fallback：检查请求路径是否以 `/api/` 开头，否则返回 `web/dist/index.html`
- `StaticDir` 配置非空时，从文件系统 serve（开发模式）；为空时使用 embed.FS（生产模式）
- Makefile 中 `make build` 先执行 `cd web && npm run build`，再执行 `go build`

# 4.6: TaskDetailPage

## Description

实现任务详情页 `/tasks/:id`，展示任务基本信息、Markdown 描述、执行记录时间线。

**组件层级：**
```
TaskDetailPage
├── PageHeader (← 返回看板)
├── TaskHeader
│   ├── TaskIdAndTitle
│   ├── StatusBadge + PriorityBadge
│   ├── ClaimedByText
│   ├── TagList
│   └── DependencyLinks (→ /tasks/:depId)
├── TaskDescription
│   └── MarkdownRenderer
└── ExecutionRecordTimeline
    ├── RecordCount
    └── RecordItem (×N, newest first)
        ├── RecordHeader (timestamp + agentId + summary + ExpandToggle)
        └── RecordDetail (collapsed by default)
            ├── FilesCreatedList
            ├── FilesModifiedList
            ├── KeyDecisionsList
            ├── TestResultsRow (✓ passed / ✗ failed / coverage%)
            └── AcceptanceCriteriaList (✓/✗ + text)
```

**执行记录懒加载：**
- 初始加载第 1 页（10 条）
- 底部显示「加载更多」按钮，点击加载下一页
- 使用 `useInfiniteQuery` 或手动分页

**返回按钮：** 返回上一个看板页（从 location.state 或 history.back()）。

**DependencyLinks：** 每个依赖 task_id 渲染为可点击链接，跳转 `/tasks/:depId`（需要先查询 task 的数字 ID）。

## Acceptance Criteria

- [ ] 任务基本信息正确展示（id、title、status、priority、tags、dependencies、claimedBy）
- [ ] Markdown 描述正确渲染（含 GFM 表格、代码块）
- [ ] 执行记录按时间倒序展示，默认折叠
- [ ] 点击「展开」显示完整记录详情
- [ ] 验收标准显示 ✓/✗ 图标
- [ ] 「加载更多」正确加载下一页记录
- [ ] 无执行记录时显示 "暂无执行记录"

## Implementation Notes

- `RecordDetail` 使用 `useState` 控制展开/折叠，默认 `false`
- `DependencyLinks` 中 task_id（如 "1.1"）需要通过 API 查询数字 ID，或直接用 task_id 字符串作为搜索参数
- 简化方案：依赖链接暂时显示为文字（V1 可接受），不做跳转
- `TestResultsRow`：`✓ {testsPassed}  ✗ {testsFailed}  Coverage {coverage}%`

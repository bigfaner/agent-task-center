# 3.1: ProjectService & FeatureService

## Description

在 `internal/service/` 实现 `ProjectService` 和 `FeatureService`，封装业务逻辑，将 DB 层数据聚合为 API 响应类型。

**ProjectService 接口：**
```go
type ProjectService interface {
    List(ctx context.Context, search string, page, pageSize int) ([]ProjectSummary, int, error)
    Get(ctx context.Context, id int64) (*ProjectDetail, error)
    Upsert(ctx context.Context, name string) (*Project, error)
}
```

`List` 需计算每个项目的 `featureCount`、`taskTotal`、`completionRate`（completed 任务数 / 总任务数 × 100）。

`Get` 返回 `ProjectDetail`，包含 `proposals`（`[]ProposalSummary`）和 `features`（`[]FeatureSummary`）。

**FeatureService 接口：**
```go
type FeatureService interface {
    ListByProject(ctx context.Context, projectID int64) ([]FeatureSummary, error)
    GetTasks(ctx context.Context, featureID int64, filter TaskFilter) ([]Task, error)
}
```

`FeatureSummary.completionRate` 同样需要聚合计算。

## Acceptance Criteria

- [ ] `ProjectService.List` 返回正确的 `completionRate`（0 任务时为 0）
- [ ] `ProjectService.Get` 返回 404 (`ErrNotFound`) 当项目不存在
- [ ] `FeatureService.GetTasks` 正确传递 `TaskFilter` 到 DB 层
- [ ] `go test ./internal/service/... -v` 使用 interface stub，覆盖率 ≥ 70%

## Implementation Notes

- Service 层依赖 DB 函数（非接口），直接调用 `internal/db` 包函数
- `sql.ErrNoRows` 已在 DB 层包装为 `ErrNotFound`，Service 层直接透传
- `completionRate` 计算：`float64(completedCount) / float64(totalCount) * 100`，totalCount=0 时返回 0

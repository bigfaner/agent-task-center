# 3.3: UploadService (parse & upsert files)

## Description

在 `internal/service/` 实现 `UploadService`，负责接收上传文件内容，调用 Parser 解析，再调用 DB 层 Upsert。

**UploadService 接口：**
```go
type UploadService interface {
    ParseAndUpsert(ctx context.Context, projectName, featureSlug, filename string, content []byte) (*UpsertSummary, error)
}
```

**文件路由逻辑（按 filename）：**
- `index.json` → `ParseIndexJSON` → Upsert Tasks（需要 featureSlug）
- `proposal.md` → `ParseProposalMD` → Upsert Proposal
- `manifest.md` → `ParseManifestMD` → Upsert Feature

**Upsert 语义：**
- Project：按 name 查找，不存在则创建
- Proposal：按 (project_id, slug) 查找，存在则更新 title/content
- Feature：按 (project_id, slug) 查找，存在则更新 name/status/content
- Task：按 (feature_id, task_id) 查找，存在则更新 title/description/priority/tags/dependencies；**不更新** status/claimed_by/version

**UpsertSummary 返回：**
```go
UpsertSummary{
    Filename: "index.json",
    Created:  5,
    Updated:  3,
    Skipped:  0,
    Message:  "新增 5 个任务，更新 3 个任务",
}
```

**Push 批量上传：**
另实现 `PushDocs(ctx, projectName string, files []UploadFile) ([]UpsertSummary, error)`，遍历文件列表，从文件路径提取 featureSlug（`docs/features/<slug>/tasks/index.json`），逐一调用 `ParseAndUpsert`。

## Acceptance Criteria

- [ ] `ParseAndUpsert("index.json", ...)` 正确创建/更新任务，不覆盖执行状态
- [ ] `ParseAndUpsert("manifest.md", ...)` 正确 Upsert Feature
- [ ] `ParseAndUpsert("proposal.md", ...)` 正确 Upsert Proposal
- [ ] 重复上传同一 index.json 幂等（第二次全部为 updated，created=0）
- [ ] `go test ./internal/service/... -v` 覆盖 Upsert 幂等性场景

## Implementation Notes

- `featureSlug` 对 `index.json` 必填，对 `manifest.md` 从解析结果获取，对 `proposal.md` 从路径提取
- 路径提取正则：`docs/features/([^/]+)/tasks/index.json`
- 文件名不在白名单内（非 index.json/proposal.md/manifest.md）返回 `ErrInvalidFile`

# 4.1: React app setup & API client

## Description

完成 React 应用的基础配置，包括路由、全局状态、API client 封装。

**路由配置（react-router-dom v6）：**
```
/                          → ProjectListPage
/projects/:id              → ProjectDetailPage
/features/:id/tasks        → FeatureKanbanPage
/tasks/:id                 → TaskDetailPage
/proposals/:id             → DocViewerPage
```

**API client（`src/api/client.ts`）：**
```typescript
const BASE_URL = import.meta.env.VITE_API_URL ?? ''

async function request<T>(path: string, options?: RequestInit): Promise<T>
```

**各资源 API 函数（`src/api/`）：**
```typescript
// projects.ts
export function listProjects(params: { search?: string; page?: number; pageSize?: number })
export function getProject(id: number)

// features.ts
export function getFeatureTasks(id: number, filter: TaskFilter)

// tasks.ts
export function getTask(id: number)
export function listTaskRecords(id: number, params: { page?: number; pageSize?: number })

// docs.ts
export function getProposalContent(id: number)
export function getFeatureContent(id: number)

// upload.ts
export function uploadFile(projectName: string, featureSlug: string | undefined, file: File)
```

**React Query 配置：** 在 `main.tsx` 中初始化 `QueryClient`，`staleTime: 30_000`。

**共享组件（`src/components/`）：** 创建占位文件：`StatusBadge`, `PriorityBadge`, `CompletionRateBar`, `RelativeTime`, `TableSkeleton`, `EmptyState`, `ErrorState`, `MarkdownRenderer`, `UploadDialog`, `AppHeader`。

## Acceptance Criteria

- [ ] `npm run dev` 启动，访问 `/` 不报错（空页面可接受）
- [ ] `src/api/` 下所有函数有正确的 TypeScript 类型定义
- [ ] React Query `QueryClient` 正确初始化
- [ ] 路由配置完整，所有路径可访问（组件可为占位）
- [ ] `npm run build` 无 TypeScript 错误

## Implementation Notes

- `VITE_API_URL` 开发时设为 `http://localhost:8080`，生产时为空（同源）
- API 函数返回 `Promise<T>`，错误时 throw（React Query 自动处理）
- TypeScript 类型定义放在 `src/types/index.ts`，与 api-handbook 的 JSON 结构对应
- `vite.config.ts` 配置 proxy：`/api` → `http://localhost:8080`（开发时）

# 4.7: DocViewerPage & UploadDialog

## Description

实现文档查看页 `/proposals/:id` 和文件上传弹窗 `UploadDialog`。

---

### DocViewerPage

双栏布局（桌面端）：左侧 Markdown 内容，右侧 TOC 目录导航。

**组件层级：**
```
DocViewerPage
├── PageHeader (← 返回)
├── DocTitle
├── DocLayout (desktop: 2-col, mobile: 1-col)
│   ├── MarkdownContent (MarkdownRenderer)
│   └── TableOfContents (sticky sidebar)
│       └── TocItem (×N, from H2/H3)
└── RelatedSection
    └── RelatedFeatureLinks (→ /features/:id/tasks)
```

**TOC 生成：** 解析 Markdown 内容，提取所有 H2/H3 标题，生成锚点链接。

**状态：** Loading → spinner；Error → "文档不存在或加载失败" + 返回按钮。

---

### UploadDialog

文件上传弹窗，支持拖拽和点击选择。

**状态机：** `Idle → Validating → Ready/Invalid → Uploading → Success/Error`

**组件层级：**
```
UploadDialog
├── DialogHeader
├── ProjectSelector (dropdown, 从 projects 列表获取)
├── DropZone (drag & drop + click)
├── ValidationMessage (error)
├── UploadProgress (indeterminate bar)
├── UploadResult (success summary)
└── DialogFooter (取消 / 上传)
```

**校验规则：**
- 非 .json/.md → "仅支持 .json 和 .md 文件"
- 超过 5MB → "文件大小不能超过 5MB"

**上传成功后：** 显示 `UpsertSummary` 摘要文字，关闭弹窗时刷新父页面数据。

## Implementation Notes

- TOC 锚点：`react-markdown` 的 `components` prop 为 h2/h3 添加 `id` 属性
- `DropZone` 使用原生 HTML5 drag events（`onDragOver`、`onDrop`）
- `ProjectSelector` 调用 `listProjects` 获取选项，`pageSize=100` 获取全部
- 上传成功后调用 `onSuccess` callback，父组件负责 invalidate query

# 4.5: FeatureKanbanPage

## Description

实现任务看板页 `/features/:id/tasks`，四列 Kanban 布局，支持服务端过滤，筛选条件序列化到 URL。

**组件层级：**
```
FeatureKanbanPage
├── PageHeader (← 返回 / Feature 名称)
├── FilterBar
│   ├── PrioritySelect (multi: P0/P1/P2)
│   ├── TagSelect (multi, 动态从任务 tags 聚合)
│   ├── StatusSelect (multi: pending/in_progress/completed/blocked)
│   └── ClearFiltersButton (有筛选时显示)
└── KanbanBoard
    ├── KanbanColumn: Pending
    ├── KanbanColumn: In Progress
    ├── KanbanColumn: Completed
    └── KanbanColumn: Blocked
        └── TaskCard (×N)
```

**URL 状态同步：**
```typescript
// 从 URL 读取初始筛选
const [searchParams, setSearchParams] = useSearchParams()
const priority = searchParams.get('priority')?.split(',') ?? []
// 筛选变更时 replaceState 更新 URL
setSearchParams(params, { replace: true })
```

**数据获取：** 筛选参数作为 query string 传给 API，服务端过滤后返回。

**TaskCard 设计：**
```
┌─────────────────────────────┐
│ 1.1  [P0 red badge]         │
│ Setup project scaffold      │
│ [core] [api]                │
│ agent-01                    │
└─────────────────────────────┘
```
点击卡片跳转 `/tasks/:id`。

**列标题：** 显示列名 + 当前列任务数（如 `Pending (5)`）。

## Implementation Notes

- `TagSelect` 的选项从当前 feature 所有任务的 tags 聚合（去重）
- 筛选参数传给 API：`priority=P0,P1&tag=core&status=pending`（逗号分隔）
- 使用 `useSearchParams` 管理 URL 状态，筛选变更用 `replace: true` 避免产生历史记录
- 看板列使用 CSS Grid 或 Flex 布局，Tablet 下横向滚动

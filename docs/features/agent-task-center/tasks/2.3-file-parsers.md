# 2.3: File parsers

## Description

在 `internal/parser/` 实现三种文件格式的解析逻辑，实现 `Parser` 接口：

```go
type Parser interface {
    ParseIndexJSON(data []byte) ([]TaskInput, error)
    ParseProposalMD(slug string, data []byte) (*ProposalInput, error)
    ParseManifestMD(data []byte) (*FeatureInput, error)
}
```

**ParseIndexJSON：**
- 解析 `index.json`，提取 `tasks` 对象中每个任务
- 校验每个任务必须有 `task_id` 和 `title`，缺失返回 `ErrInvalidFile`
- `priority` 缺省为 `P1`

**ParseProposalMD：**
- `slug` 来自文件路径目录名（调用方传入）
- `title` 解析自 Markdown 第一个 H1（`# Title`）
- `content` 为原始 Markdown 内容

**ParseManifestMD：**
- 解析 YAML frontmatter，提取 `feature`（slug）和 `status` 字段
- `name` 与 `slug` 相同（V1 简化）
- `content` 为原始 Markdown 内容

- [ ] `ParseIndexJSON` 正确解析合法 index.json，返回 TaskInput 列表
- [ ] `ParseIndexJSON` 对缺少 `task_id` 或 `title` 的任务返回 `ErrInvalidFile`
- [ ] `ParseProposalMD` 正确提取第一个 H1 作为 title
- [ ] `ParseManifestMD` 正确解析 YAML frontmatter 中的 `feature` 和 `status`
- [ ] `go test ./internal/parser/... -v` 覆盖率 ≥ 80%，含边界/错误输入

## Implementation Notes

- YAML frontmatter 解析：使用 `gopkg.in/yaml.v3` 或手写简单解析（frontmatter 格式固定）
- 提取 H1：扫描行，找到第一个以 `# ` 开头的行，去掉前缀
- index.json 的 `tasks` 字段为 `map[string]TaskInput`，key 为任意字符串（忽略 key，只用 value）
- 测试文件放在 `internal/parser/testdata/` 目录

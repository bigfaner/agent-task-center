# 2.2: Go data models & structs

## Description

在 `internal/model/` 定义所有 Go 数据结构，包括 DB 实体、Parser 输入类型、Service 聚合返回类型。

**DB 实体：** `Project`, `Proposal`, `Feature`, `Task`, `ExecutionRecord`

**Parser 输入类型：** `TaskInput`, `ProposalInput`, `FeatureInput`

**Service 聚合类型：** `ProjectSummary`, `ProjectDetail`, `ProposalSummary`, `FeatureSummary`, `TaskDetail`, `UpsertSummary`

**过滤类型：**
```go
type TaskFilter struct {
    Priorities []string
    Tags       []string
    Statuses   []string
}
```

**Sentinel errors（放在 `internal/service/errors.go`）：**
```go
var (
    ErrNotFound          = errors.New("not found")
    ErrNoAvailableTask   = errors.New("no available task")
    ErrVersionConflict   = errors.New("version conflict")
    ErrInvalidFile       = errors.New("invalid file format")
    ErrUnauthorizedAgent = errors.New("task claimed by different agent")
)
```

## Acceptance Criteria

- [ ] 所有 DB 实体 struct 字段与 schema 列名一一对应（`db:` tag）
- [ ] 所有 JSON 响应 struct 字段使用 camelCase（`json:` tag）
- [ ] `TaskInput` 包含 `task_id`、`title`、`description`、`priority`、`tags`、`dependencies`
- [ ] Sentinel errors 定义完整，`errors.Is` 可正确匹配
- [ ] `go build ./...` 无编译错误

## Implementation Notes

- `Task.Tags` 和 `Task.Dependencies` 在 DB 层存储为 JSON 字符串，在 Service 层序列化/反序列化
- `TaskDetail` 中 `Tags` 和 `Dependencies` 为 `[]string`（已反序列化）
- 所有时间字段使用 `time.Time`，JSON 序列化为 RFC3339

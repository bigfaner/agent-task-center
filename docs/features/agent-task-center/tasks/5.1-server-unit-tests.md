# 5.1: Server unit tests (parser + service)

## Description

为 `internal/parser/` 和 `internal/service/` 编写单元测试，达到覆盖率目标。

**Parser 测试（`internal/parser/parser_test.go`）：**

| 测试用例 | 描述 |
|---------|------|
| `TestParseIndexJSON_Valid` | 合法 index.json，验证所有字段正确解析 |
| `TestParseIndexJSON_MissingTaskID` | 缺少 task_id，期望 ErrInvalidFile |
| `TestParseIndexJSON_MissingTitle` | 缺少 title，期望 ErrInvalidFile |
| `TestParseIndexJSON_DefaultPriority` | 无 priority 字段，默认 P1 |
| `TestParseProposalMD_Valid` | 合法 proposal.md，验证 title/content |
| `TestParseProposalMD_NoH1` | 无 H1 标题，title 为空或文件名 |
| `TestParseManifestMD_Valid` | 合法 manifest.md，验证 slug/status |
| `TestParseManifestMD_NoFrontmatter` | 无 frontmatter，期望错误 |

**Service 测试（`internal/service/`）：**

| 测试用例 | 描述 |
|---------|------|
| `TestClaimTask_Success` | 正常认领，返回任务 |
| `TestClaimTask_NoAvailable` | 无可用任务，返回 ErrNoAvailableTask |
| `TestClaimTask_OptimisticLock` | 模拟并发抢占，验证重试逻辑 |
| `TestSubmitRecord_Success` | 正常提交，任务变为 completed |
| `TestSubmitRecord_WrongAgent` | agentId 不匹配，返回 ErrUnauthorizedAgent |
| `TestUpsertTask_Idempotent` | 重复 upsert 不覆盖 status/claimed_by |

**测试工具：** `testify/assert`，Service 测试使用 in-memory SQLite（不 mock DB）。

## Acceptance Criteria

- [ ] `go test ./internal/parser/... -v -cover` 覆盖率 ≥ 80%
- [ ] `go test ./internal/service/... -v -cover` 覆盖率 ≥ 80%
- [ ] 所有测试用例通过，无 flaky test
- [ ] 测试数据放在 `testdata/` 目录，不硬编码在测试文件中

## Implementation Notes

- Service 测试使用真实 SQLite in-memory DB（`:memory:`），不 mock
- 测试前调用 `db.RunMigrations` 初始化 schema
- 每个测试用例使用独立的 DB 实例（避免状态污染）
- `testify/assert` 用于断言，`testify/require` 用于致命断言

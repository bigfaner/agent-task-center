# 5.2: Integration tests (HTTP + concurrent claim)

## Description

使用 `net/http/httptest` 编写完整 HTTP 链路的集成测试，覆盖关键业务场景。

**测试场景：**

| 场景 | 描述 |
|------|------|
| `TestPushClaimRecord_Flow` | push index.json → claim → submit record 完整流程 |
| `TestConcurrentClaim_3Agents` | 3 个 goroutine 同时 claim，验证无冲突、每个任务只被认领一次 |
| `TestUpsert_Idempotent` | 重复 push 同一 index.json，执行记录不丢失 |
| `TestUpload_InvalidFile` | 上传非法文件，返回正确错误码 |
| `TestUpload_FileTooLarge` | 上传超 5MB 文件，返回 413 |
| `TestGetFeatureTasks_Filter` | 带 priority/tag/status 过滤，返回正确子集 |
| `TestClaimTask_NoAvailable` | 所有任务已认领，返回 404 no_available_task |
| `TestUpdateStatus_WrongAgent` | 错误 agentId 更新状态，返回 403 |

**测试基础设施：**

## Acceptance Criteria

- [ ] `go test ./... -v -run Integration` 所有集成测试通过
- [ ] 并发 claim 测试：3 个 agent 各认领 1 个不同任务，无冲突
- [ ] push → claim → record 完整流程测试通过
- [ ] 幂等性测试：重复 push 后执行记录数量不变
- [ ] `go test ./internal/handler/... -v -cover` 覆盖率 ≥ 60%

## Implementation Notes

- 使用 `sync.WaitGroup` 协调并发测试 goroutine
- 每个集成测试用独立的 httptest.Server 实例（避免状态污染）
- 测试文件命名：`*_integration_test.go`，使用 build tag `//go:build integration` 可选隔离
- 并发测试中用 `sync.Mutex` 收集结果，测试结束后统一断言
